üîó Basic Blocks and Chains

History of Blockchains

1991: Cryptographically secured chain of blocks For tamper-proof timestamps of digital documents.

1992: Merkle Tree added Hash-tree or chain with hash of previous node.

1997: Hashcash proof-of-work system Proof that compute-resources were invested.

2008: Hashcash added to the chain of blocks Bitcoin-inventors propose distributed proof-of-work chain.

2009: Bitcoin released Satoshi Nakamoto mine genesis block and publish code.

Properties of a chain of blocks

The first block is valid if its index is equal to 0, it does not have a previous has and its hash is correct.

A new block is valid w.r.t. the previous Blockchain block if its index is incremented by 1 compared to the index of the previous block, its previous hash field is filled with the hash of the previous block and its hash is coherent.

Blockchain is valid if the first block is valid and each block that composes it is valid w.r.t. to the block that precedes it.

Distributed Ledger

Peer-to-peer network ensures there is no central authority that can be compromised.

Proof-of-Stake or Voting consensus algorithms to ensure a majority of nodes agree on a transaction.

Eventually consistent to satisfy high availability in a distributed network.

Ethereum Basics

Introduced in 2015
by Vitalik Buterin, financed by crowdsale.

Ethereum Virtual Machine
to run scripts on the decentralized nodes.

Ethereum (ETH) and Ethereum Classic (ETC)
are the currencies which were split after a flaw was discovered in 2016.

Ethereum Concepts

Smart Contract
is a script that is stored in a block and runs on any network node.

Ether
is the currency used to compensate node owners for their mining work.

Gas
is the transaction fee.

Init geth

Create a new local account
geth account new

Copy your account address
into the genesis.json to init some ether.

Init your node with the genesis block
geth init genesis.json

Start geth

Run the local Ethereum node. 

geth \
  --networkid 15151 \  # Our network id.
  --rpc \	 # RPC interface enabled.
  --rpcport "8545" \  # RPC port.
  --rpccorsdomain "http://localhost:8000" \  # Allow local requests.
  --port "30303" \  # Node port.
  --nat "any"  \  # Not sure.
  --identity "{NODE_NAME}"  # Unique node name.

Run geth Commands

In a new terminal, attach to the node
geth attach

Run a few commands
eth.coinbase
eth.getBalance(eth.coinbase)
eth.accounts

Run the Utils 1/3

Adjust the app.json in eth-net-intelligence-api
For INSTANCE_NAME, pick the same as identity above.
For WS_SECRET, use ethereum.

In a new terminal, attach the net status listener
cd eth-net-intelligence-api && pm2 start app.json

Run the Utils 2/3

In a new terminal, run the net status UI
cd eth-netstats && WS_SECRET=ethereum npm start

Open the net status UI
http://localhost:3000

Run the Utils 3/3

In a new terminal, run the block explorer UI
cd eth-explorer && npm start

Open the block explorer UI
http://localhost:5000

Create transactions

Send some ether
eth.sendTransaction({from:eth.coinbase, to:'RANDOM_ADRESS', value: toWei(0.05, "ether"), gas:21000});

Check pending transactions
eth.pendingTransactions

Restart geth
with --mine --minerthreads=1

Create private network

Restart geth
with --bootnodes "enode://ENODE@IP:30303"

Send some ether to another account
eth.sendTransaction({from:eth.coinbase, to:'TO_ADRESS', value: toWei(0.05, "ether"), gas:21000});

üìù Smart Contracts


Modifiers

pure
Does not access or modify state.

view
Does not modify state.

payable
To receive ether.

Nothing
Creates a new block, requires gas.

Example: Multiplier

Solidity 

pragma solidity ^0.4.24;

contract Multiplier {
  function multiply(uint a) public pure returns(uint d) {
    // Mutiplies the input with 7.
    return a * 7;
  }
}
